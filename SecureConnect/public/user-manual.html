🔐 Đăng nhập & Xác thực

Bước 1: Truy cập ứng dụng

Mở trình duyệt và tới https://DBManager.allin1site.com

Bước 2: Đăng nhập Google

✓ Click nút "Sign in with Google"
Bạn sẽ được chuyển hướng tới trang đăng nhập Google. Nhập tài khoản Google của bạn.

Bước 3: Xác thực

✓ Hoàn tất 2-factor authentication (nếu có)
Sau khi đăng nhập Google thành công, bạn sẽ được tự động chuyển hướng vào dashboard.
💡 Mẹo: Để bảo mật, hãy đảm bảo luôn sử dụng HTTPS. Không bao giờ chia sẻ mật khẩu Google của bạn.

🔌 Tạo & Quản lý kết nối cơ sở dữ liệu

Các loại cơ sở dữ liệu được hỗ trợ

🐬 MySQL / MariaDB

Port: 3306 (mặc định)
Hỗ trợ SSL connection

🐘 PostgreSQL

Port: 5432 (mặc định)
Hỗ trợ SSL connection

🔷 SQL Server

Port: 1433 (mặc định)
Hỗ trợ Windows auth

Tạo kết nối mới

Trường thông tin Mô tả Ví dụ
Database Type Chọn loại DB hoặc "Auto" để tự động phát hiện MySQL, PostgreSQL, MSSQL
Host Địa chỉ server hoặc IP localhost, 192.168.1.10
Port Cổng kết nối 3306, 5432, 1433
Username Tên người dùng DB (tuỳ chọn) root, admin, sa
Password Mật khẩu (được mã hoá RSA client-side) ***
SSL Kích hoạt SSL nếu cần ☑ Enabled / ☐ Disabled
✓ Quy trình kết nối:
  1. Nhập thông tin kết nối
  2. Mật khẩu được mã hoá RSA trên trình duyệt (không gửi plain text)
  3. Click "Test Connection" để kiểm tra
  4. Click "Save Connection" để lưu vào hồ sơ của bạn
⚠️ Lưu ý: Mật khẩu được mã hoá với RSA-OAEP 2048-bit trước khi gửi. Tuy nhiên, luôn sử dụng HTTPS để bảo vệ thêm.

🔍 Truy cập & Quản lý dữ liệu

Khám phá cơ sở dữ liệu

📊 Danh sách bảng

Xem tất cả bảng trong database. Bấm vào bảng để xem cột và dữ liệu.

📋 Xem cột

Kiểm tra kiểu dữ liệu, khóa chính, giá trị mặc định cho mỗi cột.

🔗 Quan hệ bảng

Hiển thị các mối quan hệ giữa các bảng (Foreign Key references).

Xem & Lọc dữ liệu

✓ Xem bản ghi (Rows):
  1. Click vào bảng để xem dữ liệu
  2. Các hàng được hiển thị dưới dạng bảng với phân trang
  3. Click vào hàng để xem chi tiết đầy đủ
✓ Tìm kiếm & Lọc dữ liệu:
  1. Sử dụng Advanced Search để xây dựng điều kiện phức tạp
  2. Hỗ trợ các toán tử: =, ≠, >, <, >=, <=, CONTAINS
  3. Kết hợp nhiều điều kiện với AND/OR logic
  4. Xem SQL WHERE preview trước khi thực thi

Thực thi SQL trực tiếp

✓ SQL Editor:

Nhập trực tiếp câu lệnh SQL (SELECT, EXPLAIN) để thực thi. Hữu ích cho các truy vấn phức tạp mà Advanced Search không thể biểu diễn.

⚠️ Lưu ý: SQL Editor chỉ cho phép SELECT và EXPLAIN. Các lệnh DELETE, UPDATE, INSERT được chuyển hướng tới Bulk Edit để có thể review.

📥 Nhập & Xuất dữ liệu

Xuất dữ liệu (Export)

✓ Xuất tất cả bản ghi:
  1. Chọn bảng muốn xuất
  2. Click nút "Export" → CSV
  3. File CSV sẽ được tải về máy

Nhập dữ liệu (Import)

✓ Import CSV vào bảng:
  1. Click nút "Import"
  2. Chọn file CSV
  3. Hệ thống sẽ:
    • Tự động nhận dạng header (cột) từ CSV
    • Cho phép map các cột CSV sang cột bảng
    • Hiển thị preview 10 hàng đầu tiên
    • Bạn review rồi confirm import
💡 Mẹo CSV: Đảm bảo file CSV:
  • Có header row ở dòng đầu tiên
  • Encoding UTF-8
  • Delimiter là dấu phẩy (,) hoặc tab

🛡️ Bảo mật & Quyền riêng tư

Mã hoá mật khẩu

🔐 Client-side Encryption

Mật khẩu được mã hoá RSA-OAEP 2048-bit trên trình duyệt của bạn trước khi gửi tới server.

🔐 Server-side Vault

Trên server, mật khẩu được mã hoá AES-256-GCM với khóa sinh từ HKDF-SHA-256.

🔐 HTTPS / TLS

Tất cả traffic được bảo vệ bằng HTTPS. Certificate được quản lý bởi Cloudflare.

Quản lý phiên làm việc (Sessions)

✓ Session Timeout:

Phiên làm việc có thời gian sống là 24 giờ. Sau đó, bạn cần đăng nhập lại để tiếp tục.

Chia sẻ an toàn (Share Links)

✓ Tạo share link read-only:
  1. Click nút "Share" ở trang explorer
  2. Hệ thống sinh ra unique token
  3. Chia sẻ link với người khác
  4. Người nhận chỉ có quyền xem (read-only), không thể sửa/xoá
⚠️ Quy tắc bảo mật:
  • Không bao giờ chia sẻ share link publicly (nó chứa connection credentials)
  • Hãy yêu cầu người dùng login bằng Google account của họ
  • Share link có thể bị vô hiệu hoá bất cứ lúc nào từ phía owner

❓ Câu hỏi thường gặp

Có thể đăng nhập bằng cách khác ngoài Google không?

Hiện tại chỉ hỗ trợ Google Sign-in. Liên hệ admin nếu bạn muốn các phương thức khác.

Mật khẩu DB được lưu ở đâu?

Mật khẩu được mã hoá AES-256-GCM và lưu trong database nội bộ. Nó không bao giờ được lưu ở dạng plain text. Server key được quay vòng định kỳ để bảo mật thêm.

Có thể thay đổi mật khẩu DB từ app không?

Không. DatabaseManager chỉ đọc và truy cập dữ liệu. Để thay đổi mật khẩu DB, bạn cần truy cập trực tiếp database server.

Có giới hạn số bản ghi không?

Mặc định hiển thị 100 bản ghi/trang. Bạn có thể thay đổi limit hoặc sử dụng SQL Editor để thực thi SELECT với LIMIT/OFFSET tuỳ chỉnh.

Có thể chia sẻ read-only access với ai không?

Có. Sử dụng Share Link feature để tạo unique token. Người khác có thể xem dữ liệu mà không cần credentials cơ sở dữ liệu.

Có log hoạt động không?

Có. Tất cả hoạt động (login, query, import) được ghi log để audit. Admin có thể xem logs để kiểm tra hoạt động.

Cách liên hệ support?

Vui lòng gửi email tới admin hoặc kiểm tra trang "Features & Admin" để có thông tin liên hệ.
