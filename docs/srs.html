4. Environment Variables

Authentication & Network

  • AUTH_SECRET — NextAuth session encryption (base64, ≥32 bytes)
  • AUTH_URL — https://dbmanager.allin1site.com (must match user-facing host)
  • AUTH_GOOGLE_ID, AUTH_GOOGLE_SECRET — OAuth credentials
  • ALLOWED_EMAILS, ALLOWED_EMAIL_DOMAINS, AUTH_ALLOW_ANY — allowlist (fail-closed by default)

Database & Encryption

  • VAULT_MASTER_SECRET — Master encryption key for saved profiles (required for feature; PBKDF2 derived)
  • VAULT_MASTER_SECRET_OLD — Previous master (decrypt-only during rotation)
  • SERVER_KEYPAIR_PATH — Ephemeral or persistent RSA keypair location
  • SAVED_CONNECTIONS_PATH — Encrypted connection profiles (default: /data/saved-connections.json)
  • DB_SSL_STRICT — MySQL/PG TLS cert validation (false = self-signed OK) PG/MSSQL cert validation; MySQL now defaults opportunistic TLS
  • DB_SSL_DISABLED — Set true only for MySQL servers that cannot do TLS (default: false = TLS on)
  • MSSQL_TRUST_SERVER_CERT — MSSQL self-signed certs

Session & Connection Lifecycle

  • SESSION_MAXAGE_SEC — Session expiry (default: 86400 = 24h)
  • DB_MAX_CONCURRENT_CONNECTS — Global /api/connect cap (default: 10)
  • DB_MAX_PER_USER_CONNECTS — Per-user cap (default: 2)
  • CONNECTION_TTL_MS — In-memory record lifetime (default: 300000 = 5min) — validated with Number.isFinite; non-numeric values (e.g. "30m") now fall back to default instead of producing NaN TTL
  • CONNECTION_MAX_RECORDS — Max concurrent connections (default: 1000; LRU evict)

Connection Pooling (Wave 3 #15)

  • DB_POOL_ENABLED — Enable per-connection-id reuse (default: false = fresh connect per request)
  • DB_POOL_IDLE_MS — Idle timeout before eviction (default: 30000 = 30s)

Redis Shared State (Wave 3 #1)

  • REDIS_URL — Connection string (e.g., redis://redis:6379); unset = in-memory fallback
  • REDIS_PREFIX — Namespace for shared Redis instance (default: dbm:)

5. Security & Authorization

  • Session-Based: NextAuth JWT stored in secure httpOnly cookie
  • Per-Connection Encryption: Credentials encrypted in serverVault (HKDF + AES-256-GCM); key rotation supported
  • Rate Limiting: GET /api/crypto/public-key (30/min), row preview/mutate (30/min); Redis ZSET with pexpire auto-cleanup
  • Confirmation Tokens: Single-use, 2-min TTL, hash-bound to exact operation (prevents token-reuse attacks)
  • Access Controls:
    • ✅ Loopback check (127.0.0.1, ::1 always blocked) Loopback check — blocks compressed AND expanded IPv6 forms (e.g. 0:0:0:0:0:0:0:1)
    • ✅ Private host allowlist (10.0.0.0/8, 172.16/12, 192.168/16, fc00::/7) — gated by ALLOW_PRIVATE_HOSTS
    • ✅ Origin/Referer hardening — /api/connect validates request host matches AUTH_URL CSRF guard fail-closed: when AUTH_URL unset, all cross-origin requests are rejected (not silently allowed)
    • ✅ Optional Cloudflare hardening — REQUIRE_CLOUDFLARE=true enforces CF-Connecting-IP header
  • SQL Validator hardening: INTO OUTFILE, INTO DUMPFILE, LOAD DATA, LOAD_FILE now blocked to prevent file-system exfiltration via /api/db/[id]/query
  • LIKE metachar safety: %, _ characters in search values are escaped before LIKE interpolation; ESCAPE clause added cross-driver
  • Count cache invalidation: Row count cache (Redis) cleared after INSERT/UPDATE/DELETE to prevent stale pagination totals
  • SSH Hardening: fail2ban sshd jail (prod-106) — 1h ban after 5 failed logins in 10m; operator IP whitelisted
  • Secrets Handling: No secrets in logs, no credentials in URL params, environment-injected only

6. Deployment & Operations

Build & Test Gate

  • Vitest Framework: 129 unit tests covering rate-limit, token lifecycle, connection records
  • Build Gate: npm run build (Next.js static generation + API route validation)
  • Lint Gate: ESLint + TS strict mode; all async functions verified

Docker Compose

  • App Service (dbmanager-app):
    • Image: Node.js runtime
    • RAM limit: 512M / reservation: 128M
    • CPU limit: 1.0 cores
    • Health check: GET http://127.0.0.1:3000/api/health every 30s (3 retries, 10s timeout)
  • Redis Service (dbmanager-redis, Wave 3):
    • Image: redis:7-alpine
    • RAM limit: 192M
    • Persistence: AOF (appendonly=yes, volatile-ttl eviction)
    • Health check: redis-cli ping every 10s
    • Depends-on: App waits for redis service_healthy
  • Volumes: dbmanager_data (saved-connections.json, keypair, backups)

Deployment Flow

  • No local Node/Docker; build on server via deploy.ps1 + Dockerfile test gate
  • git pull → docker build (no cache) → container restart → health check → Cloudflare purge (if configured)
  • Safety: NEVER start nginx, NEVER bind 80/443 locally, verify 13 other services unaffected
